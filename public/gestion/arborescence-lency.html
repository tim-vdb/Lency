Vue d'ensemble

Lency est une plateforme web construite avec Next.js 16 (App Router, Turbopack) et React 19. C'est un projet scolaire modulaire qui suit une architecture stricte pour favoriser la scalabilité et la maintenance.

Principes fondamentaux

✅ Séparation des responsabilités — Chaque couche a un rôle unique (Repository → Service → API)

✅ Typage strict — Dériver toujours les types de Prisma, jamais redéfinir

✅ DRY (Don't Repeat Yourself) — Extraire l'état partagé dans des hooks, spread sur les enfants

✅ Optimistic Updates — Les mutations React Query mettent à jour le cache avant la réponse serveur

Stack technique

Frontend React 19 + TypeScript + Next.js 16 (App Router, Turbopack)
UI shadcn/ui (composants Radix UI) + Tailwind CSS 4

Notifications sonner (toasts)
Secrets Doppler CLI en développement

Patterns clés

1️⃣ Flux complet pour une nouvelle entité

Illustration avec l'entité "Project"

Repository — Data access pur

// src/back/repositories/projects.action.ts
export const ProjectAction = {
    findAll: () => prisma.project.findMany({
        include: { author: true }
    }),
    findById: (id: string) => prisma.project.findUnique({
        where: { id },
        include: { author: true }
    }),
    create: (data: Prisma.ProjectCreateInput) =>
        prisma.project.create({ data }),
    update: (id: string, data: Prisma.ProjectUpdateInput) =>
        prisma.project.update({ where: { id }, data }),
    delete: (id: string) =>
        prisma.project.delete({ where: { id } }),
};

Service — Logique métier + vérification auth

// src/back/services/projects.service.ts
export const ProjectService = {
    findAll: async () => ProjectAction.findAll(),
    create: async (data: CreateProjectInput) => {
        const user = await getUser();
        if (!user) throw new Error("Unauthorized");
        return ProjectAction.create({
            ...data,
            authorId: user.id
        });
    },
};

Route Handler — API endpoint

// src/app/api/projects/route.ts
export async function GET() {
    try {
        const projects = await ProjectService.findAll();
        return NextResponse.json({ projects });
    } catch {
        return NextResponse.json(
            { error: "Erreur serveur" },
            { status: 500 }
        );
    }
}

export async function POST(req: NextRequest) {
    try {
        const data = await req.json();
        const project = await ProjectService.create(data);
        return NextResponse.json({ project }, { status: 201 });
    } catch (error) {
        if (error instanceof Error && error.message === "Unauthorized")
            return NextResponse.json(
                { error: "Non autorisé" },
                { status: 401 }
            );
        return NextResponse.json(
            { error: "Erreur serveur" },
            { status: 500 }
        );
    }
}

Fetch wrapper — Client-side API call

// src/front/lib/api/projects.ts
export async function fetchProjects(): Promise<Project[]> {
    const res = await fetch('/api/projects', {
        cache: 'no-store'
    });
    if (!res.ok) throw new Error('Erreur fetch');
    return (await res.json()).projects;
}

React Query hook

// src/front/hooks/queries/use-projects.ts
const PROJECT_ROOT = ["projects"] as const;

export const projectQueries = {
    lists: () => queryOptions({
        queryKey: [...PROJECT_ROOT, "list"],
        queryFn: fetchProjects,
        staleTime: 1000 * 60 * 5,
    }),
};

export const useProjects = () => useQuery(projectQueries.lists());

export const useCreateProject = () => {
    const queryClient = useQueryClient();
    return useMutation({
        mutationFn: createProject,
        onSuccess: () => queryClient.invalidateQueries({
            queryKey: [...PROJECT_ROOT, "list"]
        }),
    });
};

3️⃣ Mutations — optimistic updates obligatoires

// Pattern pour toute mutation
export const useToggleVoteProject = (projectId: string) => {
    const queryClient = useQueryClient();
    const listKey = projectQueries.lists().queryKey;

    return useMutation({
        mutationFn: () => toggleVoteProject(projectId),
        onMutate: async () => {
            await queryClient.cancelQueries({ queryKey: listKey });
            const previous = queryClient.getQueryData(listKey);
            // Mise à jour optimiste du cache
            queryClient.setQueryData(listKey, (old: Project[] = []) =>
                old.map(p => p.id === projectId
                    ? { ...p, isVoted: !p.isVoted }
                    : p)
            );
            return { previous };
        },
        onError: (_err, _vars, context) => {
            if (context?.previous)
                queryClient.setQueryData(listKey, context.previous);
        },
        onSettled: () =>
            queryClient.invalidateQueries({ queryKey: listKey }),
    });
};

Flux de données détaillé

Exemple : Créer un projet

// 1. Composant client soumet le formulaire
<CreateProjectForm />

// 2. useCreateProject() appelle le fetch wrapper
const { mutate } = useCreateProject();
mutate({ title: "Mon projet", ... });

// 3. Fetch wrapper lance la requête POST
POST /api/projects
Body: { title: "Mon projet", ... }

// 4. Route Handler appelle le Service
const project = await ProjectService.create(data);

// 5. Service récupère l'utilisateur et appelle le Repository
const user = await getUser();
return ProjectAction.create({ ...data, authorId: user.id });

// 6. Repository exécute la query Prisma
await prisma.project.create({
    data: { title, authorId, ... }
});

// 7. Prisma envoie à PostgreSQL et retourne l'objet créé

// 8. Route Handler retourne le JSON
return NextResponse.json({ project }, { status: 201 });

// 9. Fetch wrapper résout la Promise

// 10. React Query (onSuccess) invalide le cache
queryClient.invalidateQueries({ queryKey: ["projects", "list"] });

// 11. Le hook useProjects() se re-fetch automatiquement

// 12. UI re-render avec le nouveau projet

En cours d'optimisation

Ces patterns existent déjà dans certains endroits mais ne sont pas encore appliqués partout :

  • ✅ Optimistic updates (certaines mutations font juste invalidateQueries)
  • ✅ Extraction de state en hooks (certains composants ont encore du state inline)
  • ✅ Typage strict Prisma (quelques any peuvent subsister)

Conseil lors des modifications : Si tu corriges du code existant qui ne suit pas ces patterns, tu peux l'améliorer au passage — mais ne refactore pas pour refactorer, seulement si c'est dans le périmètre de la tâche demandée.
